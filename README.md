## Описание программы
«Expense Tracker» — это десктопное GUI-приложение, созданное для ведения персонального финансового учета и мониторинга личных расходов. Программа помогает классифицировать траты по различным категориям, отслеживать финансовые потоки во времени, автоматически вычислять итоговую сумму издержек за конкретный выбранный период и осуществлять фильтрацию накопленной базы данных.

### Ключевые возможности проекта:
* **Графический интерфейс пользователя (GUI):** Интуитивно понятная форма ввода, выпадающие списки выбора категорий, функциональные кнопки управления и интерактивная таблица для вывода финансовых записей.
* **Строгая встроенная валидация:** Защита базы данных от пустых значений. Контроль числовых полей (сумма обязана быть строго положительным числом `> 0`). Контроль временных полей (ввод дат допускается только в строгом формате `ДД.ММ.ГГГГ`).
* **Динамический подсчёт сумм:** Приложение автоматически пересчитывает итоговую сумму затрат в реальном времени при применении любых фильтров за выбранный период.
* **Фильтрация архива:** Поиск трат по названию категории, а также выборка расходов в рамках заданного диапазона дат (например, расходы за определенную неделю или месяц).
* **Сериализация данных в JSON:** Локальная синхронизация с файлом `expenses.json` при добавлении записей и автозагрузка истории при запуске программы с обработкой исключений `try-except`.

## Инструкция по запуску

### Системные требования
Для запуска приложения необходим установленный интерпретатор **Python версии 3.6 или выше**. Дополнительные внешние библиотеки устанавливать не нужно, так как все модули входят в базовую стандартную сборку Python.

### Запуск приложения
1. Откройте терминал, командную строку Windows или специализированную консоль Git Bash внутри папки с файлами проекта.
2. Реализуйте запуск скрипта стандартной командой:
   ```bash
   python main.py
   ```

## Примеры использования (Тест-кейсы)

### Тест 1: Регистрация нового расхода
1. Внесите в элементы интерфейса следующие тестовые параметры:
   * **Сумма:** *1250.50*
   * **Категория:** *Еда*
   * **Дата:** *26.05.2026*
2. Нажмите экранную кнопку **«Добавить расход»**.
3. *Результат:* Расход зафиксирован новой строкой в интерактивной таблице, поле суммы очистилось, а текстовый индикатор внизу обновил общую сумму трат. На жестком диске сформировался файл `expenses.json`.

### Тест 2: Проверка обработки ошибочного ввода
1. Введите в поле суммы буквенный текст (например, *тысяча рублей*) или отрицательное число (например, `-500`). Нажмите **«Добавить расход»**.
2. *Результат:* Алгоритм заблокирует транзакцию, данные не попадут в отчет, и программа отобразит диалоговое окно `messagebox.showerror` с уведомлением: *«Сумма должна быть положительным числом!»*.

### Тест 3: Вычисление суммы за выбранный период дат
1. Внесите три расхода с датами: `01.05.2026`, `15.05.2026` и `25.05.2026`.
2. В блоке фильтрации укажите диапазон: «Дата С» = `10.05.2026`, «Дата ПО» = `20.05.2026`. Кликните **«Применить»**.
3. *Результат:* В таблице временно скроются записи за 1-е и 25-е числа, а текстовая метка «Итого расходов за выбранный период» отобразит сумму только той транзакции, которая была совершена 15.05.2026.
