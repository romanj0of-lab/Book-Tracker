## Описание программы
«Weather Diary» — это удобное настольное GUI-приложение, предназначенное для ведения ежедневного персонального архива наблюдений за погодой. Программа позволяет фиксировать температурные изменения, добавлять текстовые описания климатических условий, отслеживать наличие осадков, а также фильтровать накопленные данные по датам и температурным границам.

### Основные возможности проекта:
* **Графический интерфейс:** Поля ввода данных формы, выпадающие списки выбора параметров, кнопки отправки и интерактивная таблица для вывода результатов.
* **Строгая валидация входящих данных:** Автоматический контроль формата ввода дат (строго по шаблону `ДД.ММ.ГГГГ`) и валидация числового поля температуры (поддержка целых и дробных чисел, отрицательных и положительных значений).
* **Гибкие фильтры архива:** Возможность быстрого отбора погодных сводок за конкретный день, а также вывод дней, температура в которых превышала установленный лимит (например, выше `+10.0°C`).
* **Постоянное локальное хранение данных:** Автоматическое сохранение архива в файл `weather_history.json` при внесении изменений и загрузка данных при запуске приложения с обработкой исключений `try-except`.

## Инструкция по запуску

### Требования к системе
Для корректной работы программы требуется установленный интерпретатор **Python версии 3.6 или выше**. Приложение использует исключительно стандартные встроенные пакеты Python, поэтому устанавливать сторонние зависимости не нужно.

### Запуск приложения
1. Откройте командную строку, терминал Windows или консоль Git Bash в папке с файлами проекта.
2. Выполните команду запуска скрипта:
   ```bash
   python main.py
   ```

## Примеры использования (Тест-кейсы)

### Тест 1: Добавление новой корректной записи
1. Введите в форму следующие параметры:
   * **Дата:** *26.05.2026*
   * **Температура:** *18.5*
   * **Описание погоды:** *Ясно, легкий северо-западный ветер*
   * **Наличие осадков:** *Нет*
2. Нажмите кнопку **«Добавить запись»**.
3. *Результат:* Запись успешно занесена в нижнюю таблицу, поля описания и температуры очистились для удобства нового ввода. На диске создался файл `weather_history.json`.

### Тест 2: Проверка валидации на ошибочный формат даты
1. Введите в поле температуры `12`, в описание `Дождь`, а в поле даты укажите некорректный формат (например: `26-05-26` или `вчера`).
2. Нажмите кнопку **«Добавить запись»**.
3. *Результат:* Приложение заблокирует операцию записи и выведет окно с ошибкой: *«Некорректный формат даты! Используйте шаблон: ДД.ММ.ГГГГ»*.

### Тест 3: Работа температурного фильтра
1. Добавьте в дневник три тестовые записи со значениями температур: `-5`, `+8` и `+15`.
2. В панели фильтрации в поле «Температура выше (°C)» введите число `10` и нажмите **«Применить фильтр»**.
3. *Результат:* В таблице на экране останется отображаться только одна запись со значением `+15`. После нажатия кнопки **«Сбросить»** отобразятся все сохраненные строки.
