## Описание программы
«Book Tracker» — это настольное GUI-приложение, разработанное для удобного ведения и каталогизации списка прочитанных или планируемых к прочтению книг. Приложение позволяет структурировать домашнюю библиотеку, вести подсчет страниц и быстро находить нужные произведения при помощи системы фильтров.

### Основные возможности приложения:
* **Интуитивный графический интерфейс:** Поля ввода данных о книге, кнопки управления и интерактивная таблица для наглядного вывода списка.
* **Валидация входящих данных:** Защита от добавления пустых строк. Контроль типов данных — поле количества страниц принимает только целые положительные числа.
* **Фильтрация записей:** Быстрый поиск книг по жанру (по частичному совпадению) и отбор книг по минимальному объему страниц (например, книги толще 200 страниц).
* **Постоянное хранение данных (JSON):** Автоматическое сохранение информации в локальный файл `books.json` и автоматическая загрузка библиотеки при каждом запуске программы.
* **Безопасность работы (try-except):** Обработка потенциальных исключений при повреждении файлов данных или сбоях операционной системы.

## Инструкция по запуску

### Требования к системе
Для запуска приложения требуется установленный интерпретатор **Python версии 3.6 или выше**. Все используемые библиотеки (`tkinter`, `json`, `os`) являются стандартными, поэтому скачивание сторонних пакетов не требуется.

### Запуск приложения
1. Откройте консоль, командную строку или терминал Git Bash в папке с проектом.
2. Введите команду запуска скрипта:
   ```bash
   python main.py
   ```

## Примеры использования (Тест-кейсы)

### Тест 1: Успешное добавление книги в таблицу
1. Заполните форму ввода следующими тестовыми данными:
   * **Название книги:** *Капитанская дочка*
   * **Автор:** *Александр Пушкин*
   * **Жанр:** *Повесть*
   * **Количество страниц:** *160*
2. Нажмите кнопку **«Добавить книгу»**.
3. *Результат:* Книга мгновенно отобразилась новой строкой в таблице, поля ввода очистились. В корневой директории сформировался файл `books.json`.

### Тест 2: Проверка работы валидации на некорректный ввод
1. Введите в текстовые поля любую информацию, а в поле «Количество страниц» введите текстовую строку (например: *много страниц*).
2. Нажмите кнопку **«Добавить книгу»**.
3. *Результат:* Система заблокирует добавление записи и отобразит модальное окно ошибки с текстом: *«Поле 'Количество страниц' должно быть целым положительным числом!»*.

### Тест 3: Тестирование фильтрации по объему страниц
1. Добавьте в список 3 книги с разным объемом (например: 120, 250 и 430 страниц).
2. В панели фильтрации в поле «Страниц (от)» введите число `200` и нажмите кнопку **«Применить фильтр»**.
3. *Результат:* Из таблицы скроется книга объемом 120 страниц. После нажатия на кнопку **«Сбросить»** исходный список книг вернется в полном объеме.
